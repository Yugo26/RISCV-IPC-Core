因為不會C，所以一開始學得比較吃力。利用資料結構而使用的指標，開關中斷的位元運算，中斷、context switch 要存取RISC-V的暫存器等等。
耗費了我大量精神在學習的基礎操作，以為知道了什麼是指標、什麼是地址，操作傳遞資料結構或陣列地址時便手到擒拿。卻馬上在 context switch 的作業中喜提編譯錯誤。
到了期末挑戰 IPC 機制，難度更是指數上升。Mutex 的遞迴鎖與擁有權檢查，與 Message Queue 的環形緩衝區與記憶體管理，每一步都是邏輯的考驗。我學會了如何區分 User 與 Kernel Mode，理解了 System Call 如何透過暫存器傳遞參數，甚至親手修復了可能導致死鎖的邏輯漏洞。雖然過程中充滿了 Bug 與 Panic，像是 Task 憑空消失或是指標轉型錯誤，但能看著 Log 顯示 Task 在等待、拿到鎖、傳送資料時，那種成就感難以言喻。這門課讓我明白，寫 OS 不只是寫 Code，更是設計系統。現在寫出的 Mutex 和 Message Queue 還差得遠，甚至在實作時不停切到老師的範例程式碼借鑑，視窗中的 AI 對話框也從沒關閉過，但我已經能漸漸在 AI 的提示下開始構建出這些核心機制。這條路還很長，但期望能繼續迎難而上。(12/22版)
